## 📋 一、產品核心定位與使命

### 產品本質
這是一個 **「數據驅動行動的 AI 營運顧問 SaaS 平台」**，目標是將分散在各種平台的電商數據（GA4、Shopline、Cyberbiz、廣告平台）自動整合、分析，並產出具體可執行的營運建議。

### 解決的核心問題
1. **資訊碎片化**：營運人員需要在多個平台（Shopline、Cyberbiz、GA4、Meta Ads、Google Ads）來回切換查看數據
2. **分析耗時**：每週需要花數小時整理營收、流量與廣告報表
3. **缺乏行動引導**：現有報表系統僅呈現數據，無法轉譯為可執行方案
4. **缺乏標準化**：團隊缺乏標準化的營運觀察指標與行動引導

### 產品使命
**讓每位營運人員都能像擁有專屬顧問一樣**，透過自動化數據分析理解營運現況、發現問題並採取行動。

### 主要客群
- **台灣中小型電商品牌**

- 其中 40% 具備導入 SaaS 工具意願
- 平均每週花費 5–10 小時進行營運回顧

## 👥 三、核心使用者角色（Persona）

### P1：行銷人員（Marketer）
- **使用頻率**：每日登入
- **核心任務**：檢視活動表現、追蹤成效、依據 Guideline 調整行銷策略
- **關鍵價值**：想知道「哪個活動值得再加碼」、「哪個廣告需要關掉」
- **典型痛點**：手動比對廣告與營收數據，效率低；每次開會都需製作報表

### P2：營運經理（Ops Manager）
- **使用頻率**：每週登入（週一、週二）
- **核心任務**：查看全站營運現況、比對目標差距、建立行動計畫
- **關鍵價值**：想知道「哪裡掉單、為什麼」、「要給團隊什麼方向」
- **典型痛點**：報表分散於多個平台；難以找出營運下降的根本原因

### P3：品牌老闆（Business Owner）
- **使用頻率**：每月或週報閱讀
- **核心任務**：快速理解營運健康狀況、掌握決策重點
- **關鍵價值**：想知道「品牌是否成長」、「是否需要投更多資源」
- **典型痛點**：缺乏時間瞭解營運細節，只看到最終營收結果

## 🔄 四、核心價值閉環（Value Loop）

```
數據整合 → KPI 監測 → 智能建議（Guideline）→ 任務追蹤（Action Queue）→ 成效回饋 → AI 學習優化
```

### 閉環流程說明

1. **數據整合**：每日自動匯入 GA4、Shopline、Cyberbiz、廣告資料至 BigQuery
2. **KPI 監測**：系統自動監控營收、流量、轉換率、客單價等指標
3. **智能建議**：當指標異常（如下降 >10%）時，AI 自動生成 Guideline 建議
4. **任務追蹤**：使用者可將建議一鍵轉為任務，設定負責人、截止日並追蹤狀態
5. **成效回饋**：任務完成後，系統自動比對前後 KPI，計算 Impact Score
6. **AI 學習**：成效資料回饋至 Guideline 模型，讓建議越來越準確

## 🧩 五、核心功能模組

### 1. 數據整合模組（Data Integration）
- **功能**：自動整合多來源資料（Shopline / Cyberbiz / GA4 / 廣告平台），每日更新至 BigQuery
- **技術**：BigQuery + Cloud Scheduler + Dataform
- **優先級**：★★★★★（P1 - 系統核心基礎）

### 2. 營運儀表板（Dashboard）
- **功能**：以營收公式（流量 × 轉換率 × 客單價）為核心，呈現全站營運概況
- **核心視圖**：
  - 營收總覽卡
  - KPI 分解金字塔（流量、CVR、AOV）
  - 七大流量策略表
  - 轉換漏斗分析
  - 商品結構卡
- **優先級**：★★★★★（P1 - 主要使用者入口）

### 3. Guideline 智能建議模組（AI-Driven Recommendation）
- **功能**：根據 KPI 落差與事件資料，自動生成具體營運建議
- **運作邏輯**：
  - Rule-based Trigger：監測指標異常（如下降 >10%）
  - AI 文字生成：透過 BQML + OpenAI API 生成自然語言建議
  - 建議分類：流量 / 轉換 / 商品 / 會員 / 廣告
- **輸出格式**：
  ```json
  {
    "category": "轉換率",
    "insight": "轉換率下降 0.3%，主要出現在廣告流量。",
    "suggestion": "建議調整 Meta Ads 受眾，聚焦最近14天訪客。"
  }
  ```
- **優先級**：★★★★★（P1 - 系統差異化核心）

### 4. 行動任務清單（Action Queue）
- **功能**：將 Guideline 轉為任務，可設定負責人、截止日與執行狀態
- **狀態流轉**：未開始 → 進行中 → 已完成 / 延期
- **成效追蹤**：任務完成後自動計算 Impact Score（前後 KPI 比較）
- **優先級**：★★★★☆（P2 - 實現閉環行動追蹤）

### 5. 報告與通知模組（Reports & Alerts）
- **功能**：週期性生成報告摘要，透過 LINE Bot 或 Email 推播
- **自動化**：每週一 09:00 匯總上週數據並推播
- **優先級**：★★★★☆（P2 - 提升留存與黏性）

### 6. 管理與安全模組（Admin & Security）
- **功能**：使用者登入、權限控管、資料隔離
- **技術**：Firebase Auth + OAuth 2.0、BigQuery IAM、RBAC
- **優先級**：★★★★★（P1 - 必須的登入與權限保護）

## 🏗️ 六、技術架構

### 架構分層
1. **L1 - Data Source 層**：Shopline API / Cyberbiz API / GA4 Export / Ads API
2. **L2 - Staging 層**：BigQuery Dataset（raw_ 前綴）- 暫存原始資料
3. **L3 - Processing 層**：Dataform（SQLX）+ BigQuery - 資料清洗與轉換
4. **L4 - Application 層**：Next.js + Node.js + Firebase - 前端查詢與任務執行
5. **L5 - Presentation 層**：Recharts / LINE API / Report Generator - 視覺化與通知

### 技術棧
| 層級 | 技術選擇 | 說明 |
|------|---------|------|
| 前端 | Next.js + Tailwind + Recharts | 儀表板與互動區塊 |
| 後端 | Node.js + Express (API Gateway) | 提供前端查詢介面 |
| 資料層 | BigQuery + Dataform | 數據整合、轉換與指標計算 |
| AI 模組 | BigQuery ML + OpenAI API | 生成自然語言建議 |
| 認證安全 | Firebase Auth + OAuth 2.0 | 帳號管理與權限控管 |
| 通知通道 | LINE API + Gmail API | 自動推播週報 |
| 部署 | GCP Cloud Run + IAM | 容器化部署與權限隔離 |

### 資料流程
```
Shopline/Cyberbiz/GA4 API → BigQuery Staging → Dataform ETL → Processed Layer
→ Dashboard API → Next.js 前端 → 使用者互動
→ Guideline Engine → AI 生成建議 → Action Queue → 成效回饋 → AI 學習
```

## 🔐 九、資料治理與安全

### 資料治理原則
1. **一致性（Consistency）**：所有資料欄位、命名、單位與時區統一
2. **可追溯性（Traceability）**：每筆資料的來源、轉換、更新時間可被追蹤
3. **最小權限（Least Privilege）**：僅授權必要人員操作敏感資料
4. **版本可控（Version Control）**：所有資料模型、表結構與轉換邏輯皆有版本紀錄

### 資料架構層級
- **Raw Layer**：`raw_*`（原始資料，不可修改）
- **Staging Layer**：`stg_*`（清洗後暫存，具時區轉換）
- **Processed Layer**：`lv0_*`, `processed_*`（統一格式的核心分析表）
- **Analytics Layer**：`view_*`, `summary_*`（報表用聚合結果）
- **Application Layer**：`guideline_*`, `task_*`（提供系統與 AI 模型使用）

### 權限角色（RBAC）
- **系統管理員 (Admin)**：最高權限，可管理所有品牌與 IAM
- **顧問 (Consultant)**：可檢視所有客戶報表、建立 Guideline、審核任務成效
- **品牌營運人員 (Merchant Operator)**：可編輯任務、設定 KPI、檢視自家數據
- **品牌負責人 (Brand Owner)**：可檢視報告、下載週報、授權顧問存取
- **一般使用者 (Viewer)**：只讀權限

### 資料隔離策略
- **橫向隔離**：以 `brand_id` 為主要隔離鍵
- **縱向分級**：按職責劃分可見欄位與操作
- **BigQuery IAM**：每個 brand 專案獨立 Service Account 權限

## 💡 十一、核心差異化優勢

### 與傳統報表工具的差異
1. **不只是報表，而是顧問**：系統不只是呈現數據，而是會「分析 + 建議 + 行動」
2. **完整營運閉環**：從數據整合 → KPI 監測 → AI 建議 → 任務追蹤 → 成效回饋 → AI 學習
3. **AI 驅動決策**：Guideline 模組結合 Rule-based + LLM，讓建議越來越準確
4. **行動導向**：每個異常都能轉化成具體任務，並追蹤執行成效

### 競爭優勢
- **自動化程度高**：每日自動匯入數據，無需手動整理
- **AI 智能建議**：根據 KPI 異常自動產出建議，而非僅呈現數據
- **任務追蹤閉環**：建議可轉為任務，並追蹤成效，形成學習循環
- **多平台整合**：整合 GA4、Shopline、Cyberbiz、廣告平台於單一平台
