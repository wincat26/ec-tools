# 電商營運輔助工具 - MVP MRD

## 1. 產品願景 (Vision)
打造一個專為 **Shopline / Cyberbiz 賣家** 的營運輔助工具。  
MVP 聚焦於幫助賣家：  
1. 輸入營收目標  
2. 自動拆解數據缺口  
3. 提供可執行的活動建議  
4. 透過遊戲化設計，陪伴賣家持續達成目標  

---

## 2. MVP 核心模組
### 2.1 目標設定與拆解
- 使用者輸入營收目標（例：九月 100 萬）  
- 系統自動拆解：所需流量 / 轉換率 / 客單價  
- 可選同步 Shopline / Cyberbiz 後台數據  

### 2.2 缺口分析
- 計算「現況 vs 目標」的差距  
- 清楚標示：缺少多少流量？需要提升多少轉換率？客單價要提高多少？  
- 視覺化呈現（進度條、紅/黃/綠燈號）  

### 2.3 活動卡片推薦
- 系統根據缺口，自動推薦三類活動方案：  
  - **流量型**：廣告導流、快閃活動  
  - **轉換型**：折扣券、會員喚回、免運  
  - **客單價型**：組合包、滿額贈  
- 每張卡片包含：  
  - 預估影響 KPI  
  - 建議執行時間  
  - 所需資源  

### 2.4 遊戲化陪跑
- **任務系統**：拆解活動卡片 → 每日/每週任務  
- **進度條**：顯示「目標完成度」百分比  
- **徽章獎勵**：完成任務或設定活動 → 解鎖徽章  
- **驚喜機制**：達到里程碑後觸發隨機小獎勵  

---

## 3. 八角框架驅動對照
| 驅動力 | MVP 對應功能 |
|---|---|
| 使命與意義 | 「小老闆之路」目標看板 |
| 進步與成就 | 任務清單、進度條、徽章 |
| 創意與回饋 | 活動卡片推薦 + 即時回饋 |
| 擁有與佔有 | 活動紀錄、成就徽章 |
| 稀缺與焦慮 | 快閃活動卡片（含倒數提醒） |
| 不確定與好奇 | 驚喜獎勵（盲盒式獎勵） |
| 失落與逃避 | 到期提醒（未完成任務推播） |

---

## 4. 使用者旅程 (User Journey)
1. **輸入目標**：設定九月營收 100 萬  
2. **缺口分析**：系統顯示需額外 20,000 流量或轉換率提升 0.5%  
3. **活動推薦**：系統提供 3 張活動卡片，選擇其一  
4. **遊戲化陪跑**：系統指派每日任務，使用者完成後獲得徽章與驚喜獎勵  
5. **進度追蹤**：看板顯示目前完成度 70%，提醒「距離目標還差 30%」  

---

## 5. 技術範圍 (Scope)
### 前端
- React + TailwindCSS  
- 模組：目標輸入框、進度條、活動卡片 UI、任務中心  

### 後端
- Node.js (NestJS/Express)  
- 模組：目標拆解、缺口分析、活動推薦 API  
- Redis：快閃活動倒數  

### 資料庫
- PostgreSQL：用戶、目標、活動、任務  
- MongoDB：活動紀錄與任務日誌  

---

## 6. MVP 成功指標 (Success Metrics)
- **功能採用**：90% 使用者成功輸入目標  
- **缺口理解**：80% 使用者能看懂缺口報表  
- **活動採用**：60% 使用者至少使用 1 張活動卡片  
- **遊戲化參與**：70% 使用者完成至少 1 個任務並解鎖徽章  
- **留存效果**：D7 留存率 ≥ 50%  

---

## 7. 非目標 (Out of Scope for MVP)
- 不整合廣告投放工具  
- 不提供完整會員 CRM 系統  
- 不做跨平台數據大整合（僅限 Shopline / Cyberbiz 基礎數據同步）  
- 不做行動端 App，MVP 以 Web 為主  

---

## 8. 里程碑 (Milestones)
- **M0**：目標輸入 + 拆解 MVP  
- **M1**：缺口分析 + 活動卡片  
- **M2**：任務系統 + 進度條  
- **M3**：徽章 + 驚喜獎勵
