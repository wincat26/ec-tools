## 🎯 一、產品願景（Vision）

### 核心願景
**打造台灣第一個「數據驅動行動的 AI 營運顧問平台」**，幫助品牌在**不增加人力**的前提下，提升決策效率與營收表現。

### 願景宣言
> 讓「每一週的營運檢視」變成「下一步的行動策略」。  
> 讓每位營運人員都能像擁有專屬顧問一樣，透過自動化數據分析理解營運現況、發現問題並採取行動。

## 🧭 二、發展脈絡（Development Context）

### 2.1 問題起源
在現今的電商營運環境中，品牌營運人員面臨以下核心痛點：

1. **資訊碎片化**
   - 需要在多個平台（Shopline、Cyberbiz、GA4、Meta Ads、Google Ads）來回切換
   - 數據分散，難以統整分析

2. **分析耗時**
   - 每週需要花數小時整理營收、流量與廣告報表
   - 缺乏自動化工具，人工成本高

3. **缺乏行動引導**
   - 現有報表系統僅呈現數據，無法轉譯為可執行方案
   - 知道「哪裡有問題」，但不知道「該怎麼做」

4. **缺乏標準化**
   - 團隊缺乏標準化的營運觀察指標與行動引導
   - 決策依賴個人經驗，缺乏數據驅動

### 2.2 市場洞察
- **約 70%** 的中小電商每週需手動整合報表
- **其中 40%** 具備導入 SaaS 工具意願
- **平均每週花費 5–10 小時**進行營運回顧
- **多數中小型電商品牌**（年營收 < 1000 萬）沒有專職數據分析人員

### 2.3 產品定位演進
1. **初期構想**：電商營運輔助工具（MVP MRD）
   - 目標設定與拆解
   - 缺口分析
   - 活動卡片推薦
   - 遊戲化陪跑

2. **進化方向**：AI 營運顧問系統
   - 從「輔助工具」升級為「AI 顧問」
   - 從「手動輸入目標」升級為「自動數據整合」
   - 從「活動推薦」升級為「智能建議 + 任務追蹤」

## 🎯 四、願景與目標（Vision & Goals）

### 4.1 核心使命（Mission）
**讓每位營運人員都能像擁有專屬顧問一樣**，透過自動化數據分析理解營運現況、發現問題並採取行動。

### 4.2 產品定位（Product Positioning）

| 面向 | 定義 |
|------|------|
| **產品類型** | SaaS 型 B2B 電商營運輔助工具 |
| **核心價值** | 數據自動整合 → KPI 監測 → 行動建議（Guideline）→ 任務追蹤（Action Queue） |
| **差異化** | 不只是數據報表，而是具備「分析 + 建議 + 行動」的完整營運閉環 |
| **使用模式** | 雲端登入平台（Web App），後端以 BigQuery 為核心資料倉儲 |

### 4.3 目標市場（Target Market）

#### 主要客群
- **台灣中小型電商品牌**
  - 月營收：70–300 萬
  - 日訪客：300–1,000
  - 使用平台：Shopline / Cyberbiz
  - 團隊規模：3–10 人，無專職數據分析人員

#### 使用者角色（Personas）
1. **行銷人員（Marketer）**
   - 使用頻率：每日登入
   - 核心需求：檢視活動表現、追蹤成效、依據 Guideline 調整行銷策略

2. **營運經理（Ops Manager）**
   - 使用頻率：每週登入（週一、週二）
   - 核心需求：查看全站營運現況、比對目標差距、建立行動計畫

3. **品牌老闆（Business Owner）**
   - 使用頻率：每月或週報閱讀
   - 核心需求：快速理解營運健康狀況、掌握決策重點

### 4.4 核心價值鏈（The Value Chain）

我們的核心價值主張，是圍繞著 **「Data → Insight → Action」** 這個價值鏈來打造的。這是我們產品為客戶創造價值的核心邏輯：

```
Data (數據彙整) → Insight (洞察生成) → Action (行動建議/自動化)
```

#### 價值鏈三層次

**1. Data（數據彙整）**
- **客戶問題**：數據分散在電商後台、GA4、Google/Meta 廣告... 需要開多個視窗才能拼湊出全貌
- **我們提供**：自動化的數據整合，這是基礎建設（Infrastructure）
- **技術實現**：BigQuery 資料倉儲 + API 串接（GA4、Shopline、Cyberbiz、廣告平台）

**2. Insight（洞察生成）**
- **客戶問題**：看到數據了，但 SO WHAT? 轉換率掉了 10%，不知道「為什麼」，也不知道「該先看哪一個」
- **我們提供**：AI 驅動的關鍵洞察，這是我們切入市場的「核心價值」
- **核心能力**：主動「推播（Push）」最重要的「關鍵課題」，並提供 AI 生成的歸因分析

**3. Action（行動建議/自動化）**
- **客戶問題**：知道問題出在 Meta 廣告了，然後呢？該換文案？該加預算？還是該停止？
- **我們提供**：可執行的「行動」，這是我們的「延伸價值」與「護城河」
- **未來能力**：AI 文案產生器、活動產生器、自動化工作流（Workflow）

### 4.5 核心價值閉環（Value Loop）

基於價值鏈，我們建立完整的營運閉環：

```
數據整合 → KPI 監測 → 智能建議（Guideline）→ 任務追蹤（Action Queue）→ 成效回饋 → AI 學習優化
```

**閉環流程說明**：
1. **數據整合（Data）**：每日自動匯入 GA4、Shopline、Cyberbiz、廣告資料至 BigQuery
2. **KPI 監測**：系統自動監控營收、流量、轉換率、客單價等指標
3. **智能建議（Insight）**：當指標異常（如下降 >10%）時，AI 自動生成 Guideline 建議
4. **任務追蹤（Action）**：使用者可將建議一鍵轉為任務，設定負責人、截止日並追蹤狀態
5. **成效回饋**：任務完成後，系統自動比對前後 KPI，計算 Impact Score
6. **AI 學習**：成效資料回饋至 Guideline 模型，讓建議越來越準確

#### 🎯 階段一：點（The Point）— AI 洞察日報（MLP）【優先開發】

**價值鏈專注**：Data → Insight  
**策略目標**：成為客戶每日決策的起點  
**對應構想**：每日 Google Chat/LINE 報告  
**戰略地位**：**當前最優先開發項目**，是整個產品策略的基礎與切入點

**核心功能**：
- 每日自動化推播（Google Chat / LINE）
- 關鍵指標彙整（業績、CVR、ROAS...）
- **[核心中的核心]** 1-3 句由 AI 生成的「關鍵課題」與「歸因」

**溝通論述**：
> 「我們不是報表工具，我們是您的『隨身 AI 數據分析師』，每天早上 9 點告訴您昨天最重要的一件事。」

**成功指標**：
- 日報開啟率 ≥ 80%
- 點擊「深入分析」轉換率 ≥ 30%
- 使用者回饋：AI 洞察準確度 ≥ 4.0 / 5

**開發重點**：
1. **先做好數據整合（Data）**：確保能穩定取得 GA4、Shopline、Cyberbiz、廣告平台數據
2. **聚焦 AI 洞察生成（Insight）**：這是階段一的核心價值，必須做到「準確、有用、可行動」
3. **建立推播機制**：確保每日能自動推播，形成使用者習慣
4. **建立連結機制**：為後續「線（儀表板）」建立入口，但先不深入開發

**時間規劃**：Q4 2025 - Q1 2026（**當前重點**）  
**完成標準**：日報開啟率 ≥ 80%，AI 洞察準確度 ≥ 4.0 / 5，才能進入階段二

#### 📊 階段二：線（The Line）— 智慧營運儀表板

**價值鏈實現**：Data → Insight → Action（建議）  
**策略目標**：成為客戶解決問題的主要平台  
**對應構想**：日報、週報、月報、AI 資料分析功能

**核心流程（User Journey）**：
1. 客戶在「點（AI 日報）」上收到 Insight：「Meta 廣告活動 A 的 ROAS 下降 30%」
2. 客戶點擊日報上的「深入分析」連結
3. 連結將他導向我們的「線（智慧儀表板）」

**核心功能**：

1. **互動式儀表板（Data 層）**
   - 日報、週報、區間查詢
   - 多維度數據分析

2. **AI 洞察分析室（Insight 層）**
   - 客戶可主動提問（例如：「幫我分析這檔活動失敗的主因」）
   - 深度歸因分析
   - 異常偵測與警示

3. **AI 行動產生器（Action 層）**
   - 基於洞察，系統推薦「行動」
   - 例如：
     - (Insight) 廣告素材疲勞 → (Action) 觸發「AI 文案/活動產生器」來生成新素材
     - (Insight) 特定會員群體流失 → (Action) 觸發「會員經營」腳本建議

**成功指標**：
- 週活躍使用率 ≥ 70%
- Guideline 採納率 ≥ 50%
- 任務完成率 ≥ 60%

**時間規劃**：Q1 2026 - Q2 2026（**待階段一完成後**）  
**啟動條件**：階段一達到成功指標（日報開啟率 ≥ 80%，AI 洞察準確度 ≥ 4.0 / 5）

#### 🌐 階段三：面（The Plane）— SMB 營運自動化平台

**價值鏈實現**：Data → Insight → Automated Action（自動化）  
**策略目標**：成為客戶的「虛擬營運團隊」，建立強大的生態系  
**對應構想**：文案/活動產生器的終局  
**戰略地位**：長期願景，待階段一、二驗證成功後啟動

**核心功能**：

## 🔧 六、要解決的問題（Problems to Solve）

### 6.1 核心問題（Core Problems）

#### 問題 1：資訊碎片化
- **現況**：營運人員需要在多個平台（Shopline、Cyberbiz、GA4、Meta Ads、Google Ads）來回切換
- **解決方案**：自動整合所有數據來源至 BigQuery，在單一平台呈現
- **驗證指標**：數據匯入成功率 ≥ 95%

#### 問題 2：分析耗時
- **現況**：每週需要花數小時整理營收、流量與廣告報表
- **解決方案**：自動化數據整合與 KPI 監測，即時呈現營運狀況
- **驗證指標**：使用者平均節省 ≥ 50% 分析時間

#### 問題 3：缺乏行動引導
- **現況**：現有報表系統僅呈現數據，無法轉譯為可執行方案
- **解決方案**：AI 驅動的 Guideline 建議模組，自動生成具體行動建議
- **驗證指標**：Guideline 採納率 ≥ 50%

#### 問題 4：缺乏標準化
- **現況**：團隊缺乏標準化的營運觀察指標與行動引導
- **解決方案**：建立標準化的 KPI 監測框架與行動建議模板
- **驗證指標**：使用者滿意度 ≥ 4.3 / 5

### 6.2 關鍵假設與驗證（Key Assumptions）

| 假設 | 驗證指標 | 驗證方式 |
|------|---------|---------|
| **使用者會採納 AI 建議** | ≥ 50% 採納率 | 系統監控：任務建立數 / 建議數 |
| **採納建議會改善營運** | ≥ 50% 任務有正向改善 | 成效追蹤：前後 KPI 比較 |
| **使用者會持續使用** | ≥ 70% 週活躍使用率 | GA4 事件分析 |
| **營運改善會帶來營收成長** | ≥ 10% 營收成長 | 比對前後三週營收平均 |

## 🎨 七、核心差異化優勢（Competitive Advantages）

### 7.1 與傳統報表工具的差異

1. **不只是報表，而是顧問**
   - 系統不只是呈現數據，而是會「分析 + 建議 + 行動」
   - 從「數據呈現」到「行動引導」

2. **完整營運閉環**
   - 從數據整合 → KPI 監測 → AI 建議 → 任務追蹤 → 成效回饋 → AI 學習
   - 形成持續優化的學習循環

3. **AI 驅動決策**
   - Guideline 模組結合 Rule-based + LLM
   - 讓建議越來越準確

4. **行動導向**
   - 每個異常都能轉化成具體任務
   - 並追蹤執行成效

### 7.2 競爭優勢

- **自動化程度高**：每日自動匯入數據，無需手動整理
- **AI 智能建議**：根據 KPI 異常自動產出建議，而非僅呈現數據
- **任務追蹤閉環**：建議可轉為任務，並追蹤成效，形成學習循環
- **多平台整合**：整合 GA4、Shopline、Cyberbiz、廣告平台於單一平台

## 📋 八、開發原則與決策框架（Development Principles）

### 8.1 核心原則

#### 原則 1：數據驅動決策
- **說明**：所有功能開發都應基於數據驗證，而非個人假設
- **應用**：每個功能都應有明確的成功指標與驗證方式

#### 原則 2：使用者價值優先
- **說明**：優先開發能直接解決使用者痛點的功能
- **應用**：功能優先級排序應以「解決問題的緊急程度」為準

#### 原則 3：完整閉環設計
- **說明**：每個功能都應考慮完整的「數據 → 分析 → 建議 → 行動 → 回饋」閉環
- **應用**：避免只做「數據呈現」而沒有「行動引導」

#### 原則 4：可擴展架構
- **說明**：技術架構應支援未來多品牌、多租戶的需求
- **應用**：以 BigQuery 為核心，設計可擴展的資料模型

### 8.2 決策框架

當面臨功能開發決策時，應考慮：

1. **是否符合產品願景？**
   - 是否幫助「數據驅動行動」？
   - 是否解決「資訊碎片化」或「缺乏行動引導」？

2. **是否形成價值閉環？**
   - 是否有「數據 → 分析 → 建議 → 行動 → 回饋」的完整流程？

3. **是否可驗證？**
   - 是否有明確的成功指標？
   - 是否有驗證方式？

4. **是否可擴展？**
   - 是否支援未來多品牌、多租戶的需求？

## 🚨 九、開發過程中的注意事項（Development Guidelines）

### 9.1 不要偏離的方向

#### ❌ 不要做的
1. **不要只做數據呈現**
   - 避免只做 Dashboard 而沒有 Guideline 建議
   - 避免只做報表而沒有行動引導

2. **不要忽略閉環**
   - 避免只做建議而沒有任務追蹤
   - 避免只做任務而沒有成效回饋

3. **不要過度複雜化**
   - 避免在 MVP 階段做太多功能
   - 優先完成核心價值閉環

4. **不要忽略使用者體驗**
   - 避免技術導向而忽略使用者需求
   - 確保「3 秒理解原則」：登入後 3 秒內可理解營運狀況

#### ✅ 應該做的
1. **優先完成核心閉環**
   - 數據整合 → KPI 監測 → Guideline 建議 → 任務追蹤 → 成效回饋

2. **確保可驗證**
   - 每個功能都應有明確的成功指標
   - 建立監控與分析機制

3. **保持架構可擴展**
   - 以 BigQuery 為核心
   - 設計支援多品牌、多租戶的資料模型

4. **持續優化 AI 建議**
   - 建立成效回饋機制
   - 讓 Guideline 建議越來越準確

### 9.2 關鍵提醒

#### 提醒 1：核心價值是「行動引導」，不是「數據呈現」
- 如果功能只做數據呈現，沒有行動建議，應重新評估優先級

#### 提醒 2：完整閉環比單點功能更重要
- 寧可先完成一個完整的閉環，也不要做多個不完整的功能

#### 提醒 3：使用者價值優先於技術炫技
- 技術選擇應以「解決問題」為準，而非「技術先進性」

#### 提醒 4：可驗證比完美更重要
- MVP 階段應優先完成可驗證的核心功能，而非追求完美

## 📝 十、總結（Summary）

### 10.1 核心要點

1. **產品願景**：打造台灣第一個「數據驅動行動的 AI 營運顧問平台」

2. **核心價值鏈**：Data → Insight → Action
   - **Data**：自動化數據整合（基礎建設）
   - **Insight**：AI 驅動的關鍵洞察（核心價值）
   - **Action**：可執行的行動建議/自動化（延伸價值與護城河）

3. **戰略三階段**：點 → 線 → 面
   - **點（階段一）**：AI 洞察日報（MLP）- 專注 Data → Insight
   - **線（階段二）**：智慧營運儀表板 - 實現 Data → Insight → Action（建議）
   - **面（階段三）**：SMB 營運自動化平台 - 實現 Data → Insight → Automated Action

4. **完整閉環**：數據整合 → KPI 監測 → Guideline 建議 → 任務追蹤 → 成效回饋 → AI 學習

5. **目標市場**：台灣中小型電商品牌（月營收 70–300 萬）

6. **成功指標**：Guideline 採納率 ≥ 50%，營收提升率 ≥ 10% / 季

### 10.2 開發指引

在開發過程中，應時時提醒自己：

- ✅ **是否符合產品願景？**
- ✅ **是否形成價值閉環？**
- ✅ **是否可驗證？**
- ✅ **是否可擴展？**

### 10.3 關鍵成功因素

1. **優先完成階段一（點）**：先做好「AI 洞察日報」，證明「Insight」的價值，再推進其他環節
2. **確保可驗證**：每個功能都應有明確的成功指標與驗證方式
3. **保持使用者價值優先**：優先開發能直接解決使用者痛點的功能
4. **階段性驗證**：只有當階段一達到成功指標（日報開啟率 ≥ 80%，AI 洞察準確度 ≥ 4.0 / 5），才進入階段二
5. **持續優化 AI 建議**：建立成效回饋機制，讓建議越來越準確
