1️⃣ 專案背景（Project Background）

在現今的電商營運環境中，品牌營運人員需要同時監控多個來源的數據（Shopline、Cyberbiz、GA4、Meta、Google Ads、CRM 等），
這造成了資訊碎片化、分析耗時、判斷不一致等問題。

現況痛點：

每週需要花數小時整理營收、流量與廣告報表。

團隊缺乏標準化的營運觀察指標與行動引導。

多數中小型電商品牌（年營收 < 1000 萬）沒有專職數據分析人員。

現有報表系統僅呈現數據，無法轉譯為可執行方案。

2️⃣ 專案使命與願景（Mission & Vision）

使命（Mission）

讓每位營運人員都能像擁有專屬顧問一樣，透過自動化數據分析理解營運現況、發現問題並採取行動。

願景（Vision）

打造台灣第一個「數據驅動行動的 AI 營運顧問平台」，
幫助品牌在 不增加人力 的前提下，提升決策效率與營收表現。

3️⃣ 產品定位（Product Positioning）
面向	定義
產品類型	SaaS 型 B2B 電商營運輔助工具
核心價值	數據自動整合 → KPI 監測 → 行動建議（Guideline）→ 任務追蹤（Action Queue）
差異化	不只是數據報表，而是具備「分析 + 建議 + 行動」的完整營運閉環
使用模式	雲端登入平台（Web App），後端以 BigQuery 為核心資料倉儲
4️⃣ 核心目標（Business Objectives）
目標項目	說明	成功指標 (KPI)
自動化數據整合	將 GA4、Shopline、Cyberbiz、廣告資料每日匯入 BQ	匯入成功率 ≥ 95%
KPI 即時監控	登入即可檢視營收、流量、轉換率、客單價	週活躍使用率 ≥ 70%
Guideline 智能建議	系統能根據指標落差自動產出建議	採納率 ≥ 50%
行動追蹤與回饋	將建議轉為任務，追蹤執行成果	任務完成率 ≥ 60%
營運績效提升	透過自動化洞察，提升整體營收	目標商家平均營收提升 ≥ 10% / 季
5️⃣ 商業價值與市場機會（Market Opportunity）
🎯 目標市場

台灣月營收 70–300 萬、日訪客 300–1000 的中小型電商品牌

使用 Shopline / Cyberbiz 為主要平台的品牌

無專職分析人員，但具備行銷／營運人力的團隊

📊 市場需求洞察

約 70% 的中小電商每週需手動整合報表

其中 40% 具備導入 SaaS 工具意願

平均每週花費 5–10 小時進行營運回顧
→ 市場痛點： 缺乏將數據自動轉化為行動的工具。

💰 商業價值
利害關係人	可獲得價值
品牌主／老闆	快速掌握營運健康度，不必參與冗長報告會議。
營運經理	系統自動產出報告與行動清單，節省人工分析時間。
行銷人員	以數據為依據執行活動，有具體方向可參考。
顧問與代營運公司	可用作「自動診斷工具」，快速回顧客戶表現。

7️⃣ 專案邊界（Scope & Constraint）
項目	定義
範圍內	GA4 + Shopline + Cyberbiz 數據整合、Dashboard、Guideline 建議、Action Queue
範圍外（MVP階段不做）	實際廣告操作功能（如 Meta API 寫入）、會員通知自動發送、KPI 自訂化
技術限制	以 BigQuery 為單一資料源，初期不支援多租戶動態分庫
時間限制	MVP 預計開發週期 8–10 週，需在 Q4 完成測試 Demo

✅ 摘要（Executive Summary）

本產品旨在成為「懂電商營運的 AI 顧問」，
將營運數據自動轉譯成具體行動建議，
協助品牌以更低成本、更高效率推動營收成長。

以數據為核心，讓「每一週的營運檢視」變成「下一步的行動策略」。
